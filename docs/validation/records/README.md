# Validation Records

Store finalized review records here.

## Naming

Use:

- `T2-<commit>.md`
- `T3-<commit>.md`

Examples:

- `T2-5bd12aa.md`
- `T4-abcdef1.md`

## Purpose

These files make review outcomes durable and auditable.
Do not rely only on chat history for final validation outcomes.

## Source of truth rule

A finalized review should be reflected in both:

- `docs/tasks/status-board.md`
- a record file in this directory
